# read doc.md


## 指导原则
- 本项目, 优先使用svelte最新语法, 优先使用.ts
- 大文件应该合理拆分, 避免文件过大
- 代码要高内聚低耦合, 但是也要避免过度封装, 清晰易懂优先才是好代码
- 尽量运用ts类型系统提高可维护性, 以及修复静态编译报错, 但是类型系统不要搞复杂, 越简单清晰越好
- 如果方案的某些细节和实际情况产生了矛盾, 可以和用户确认再做决定
- 必须坚持svelte5的语法, 无论什么情况下都不要用旧语法, 坚持最佳实践, 如果有问题可以搜索相关语法细节

## 阅读 docs/ 文档
- 编写svelte代码前, 先阅读docs\svelte.txt, 确认相关语法细节, 尤其是最新版本的语法变化
  - 当前svelte已经是最新版本
- 编写路由代码前, 先阅读docs\Universal Router.txt, 确认相关语法细节
  - 当前Universal Router已经是最新版本

## context7 mcp 工具
- 尝试用context7 mcp工具, 查找相关文档的最新语法细节

## playwright mpc 工具
- 可以用 playwright mpc 工具去搜索相关语法细节
- 阶段性完成代码后, 可以调用这个mcp工具, 可以访问 `http://localhost:5173/` 确认当前页面实现状态

## 顺序
- 先阅读 docs/ 目录下的文档文件, 确认相关语法细节
- 如果没有, 再用 context7 mcp工具去搜索相关文档
- 如果还是不够, 再用 playwright mpc 工具去搜索相关资料
